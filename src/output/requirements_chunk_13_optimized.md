```markdown
| 序号 | 模块               | 需求点                                                                 |
|------|--------------------|------------------------------------------------------------------------|
| 1    | 应用系统的可靠性   | 偶然故障率要求：应用系统在偶然故障期内以每1000小时的故障数为单位时间的故障数不得大于5。 |
| 2    | 应用系统的可靠性   | 均无故障时间（MTBF）要求：应用系统在相继两次失效之间正常工作的平均统计时间。            |
| 3    | 应用系统的可靠性   | 代码缺陷密度（FD）要求：应用软件每千行源代码（不含注释行）为一个单位，源代码中隐藏的缺陷数量应低于5个。 |
| 4    | 可用性             | 各功能在正常的业务时间内，保证的可用时间是每周的周一到周五，每天的8点到17点，即7×8的服务。 |
| 5    | 可用性             | 系统在保证可用时间内，可接受的停机时间是每次不超过30分钟，每月不超过1次。                |
| 6    | 可用性             | 当系统所在场所或整个系统发生灾难时，可接受的系统恢复服务时间是4小时。                    |
| 7    | 可用性             | 当系统的关键硬件发生灾难导致系统服务或数据不可用时，可接受的系统恢复时间目标是12小时。      |
| 8    | 可用性             | 系统发生灾难后，系统完全恢复到正常的可用性和性能水平的时间目标是3天。                    |
| 9    | 可移植性           | 本系统要求能在Windows和Linux平台之间进行移植，除去文件转化模块外，其他均可以在Linux系统运行，并可以进行容器化改造。 |
```
